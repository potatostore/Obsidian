# Heap : 완전이진트리를 목표로 채워지는 트리

Max heap : root가 가장 큰 것
Min heap : root가 가장 작은 것

시간복잡도, 공간복잡도를 줄이는 방법 -> PQ, 구간합

#### PQ(Priority Queue)
max heap, min heap을 통해 최소, 최대를 log n의 시간으로 구함

#### 구간합
index = 3~5까지의 배열의 합을 구할 때, S5 - S3을 통해 구간합을 빠르게 구할 수 있음
2차원 배열 또한 가능

---

그래프 문제 -> bfs, dfs -> 인접 배열/인접 리스트를 통해 입력값을 데이터화 함(대부분 인접 리스트로 진행)

이때 만약 노드의 값이 10^9 가 되면 arr[10^9]가 필요하고, 이는 공간복잡도가 말도 안되게 높아짐. -> 좌표 압축(re-numbering)을 통해 공간복잡도를 크게 줄임

이를 *Shorten Time Tech*라고 함. -> 현대, 카카오에서 많이 내는 코딩 문제이다.

1. 좌표, 질의의 모든 포인트를 set에 넣고
2. mapper을 통해 re-numbering

---

좌표평면 위에 여러 선이 존재하고, 가장 많은 선분이 지나는 좌표를 확인하는 방식
-> counting array를 통해 확인함

1. start, end를 설정하여 start ~ end array 만들기
2. arr[start] = 1, arr[end] = -1을 통해 구간 표시 -> sum이 0이 아닌 부분부터 0인 부분까지가 하나의 부분

---
Two pointer : 부분수열이 존재한다거나, 중복값이 존재하는 등 비교군이 존재하고, 수열 내에 두 개의 포인터를 통해 조건을 비교해가며 값을 얻을 때 유리.
