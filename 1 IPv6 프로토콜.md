기존 32비트의 주소 공간을 지원하는 IPv4체계의 한계를 보완하기 위해 IPv6가 등장했다.



**특징**

**-주소 공간 확장**

주소를 표시하는 공간이 32비트->128비트로 확장



**-헤더 구조 단순화**

기존 체계는 과거의 환경에 맞춰져있어 비효율적이다.

\->불필요한 필드 제외, 확장 헤더 형식으로 변경



**-흐름 제어 기능 지원**

흐름 제어 기능을 지원하는 Flow Label(필드)를 도입해 데이터 흐름을 지원



**IPv6 헤더 구조**

9개의 기본 필드를 지원한다.

총 40바이트 중 32바이트는 주소 공간으로 할당되고, 8바이트는 프로토콜 기능을 위해 사용한다.



**기본 헤더**

Version Number, DS, ECN, Flow Label이 있으며 바로 뒤에 확장 헤더를 둘 수 있다.



**확장 헤더**

-**Hop-by-Hop Options Header**

Jumbo 페이로드 옵션과 라우터 긴급 옵션 등과 같은 hop-by-hop 옵션을 지원한다.



**-Routing Header**

IPv4의 소스 라우팅과 유사한 기능을 제공하는데, 패킷이 Routing Header에 지정된 특정 노드를 경유하게

한다. 



**-Fragment Header**

패킷 분할과 관련된 정보를 포함한다.



**-Destination Options Header**

수신 호스트가 확인 가능한 옵션 정보



**-Authentication Header**

패킷 인증과 관련된 기능을 제공



**-Encapsulating Security Payload Header**

프라이버시 기능을 제공하기 위해 페이로드 암호화

목적지 호스트에서 암호화를 해독할 수 있는 정보도 같이 제공



**-DS/ECN 필드**

차등 서비스가 도입되면서 6비트의 DS 필드와 2비트의 ECN 필드가 정의되었다.



**-Flow Label 필드**

음성이나 영상같은 스트리밍 서비스를 제공하기 위해 존재하는 필드이다.

라우터가 중개할 때 동일한 기준을 적용해 처리할 수 있게 한다.

IPv6는 특정 송수신 호스트 사이에 전송되는 데이터를 하나의 흐름으로 정의해

중간 라우터가 특별한 기준으로 처리할 수 있게 한다.

* 이때, Flow Label을 지원하지 않는 호스트나 라우터에서는 IPv6 패킷을 생성할 때 반드시 0으로 지정한다.
* Flow Label 필드의 값이 0 이외의 동일한 번호로 부여받은 패킷은 DA, SA, Priorty, HopbyHopOtion..

등등을 모두 동일하게 지정



**-기타 필드**

Version Number: IP 프로토콜의 버전 정보

PayLoad Length / Next Header: 기본 헤더 다음에 이어지는 헤더의 유형을 수신 호스트에 알려준다.

Next Header에 표시할 수 있는 헤더는 IPv6의 확장 헤더이거나, 상위 계층인 TCP, UDP의 헤더일 수 있다.

Hop Limit: IPv4의 Time To Live를 의미

SA DA



**주소 표현**

간단하게 FFFF:FFFF:FFFF:FFFF로 표현된다.



**주소 변환**

기존 IPv4 환경을 사용하는 시스템에서 IPv6 환경을 사용하는 시스템들이 추가되면서

이 둘을 잇는 변환 기법이 필요하게 되었다.

따라서 IPv4와 함께 사용하는 환경에서 IPv4 주소를 캡슐화해서 표현한다.

X:X:X:X:X:X:d.d.d.d

X는 16비트이고 d는 8비트이다.(총 128비트)



**주소 공간**

IPv6에서는 주소 공간을 주소대별로 다른 용도로 사용한다.

상위 8비트가 0000 0000으로 시작하는 첫 번째 예약 공간에는 IPv4의 주소 공간도 포함된다.
