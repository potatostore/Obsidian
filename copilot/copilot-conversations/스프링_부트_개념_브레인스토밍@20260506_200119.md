**ai**: ## Spring Boot 개념 브레인스토밍

### 🚀 Spring Boot

*   **핵심:** Java 기반 애플리케이션 백엔드 구축 프레임워크
*   **목표:** Spring Application API 구축 지원, 보일러플레이트 코드 감소

---

### 💡 핵심 개념

*   **Spring Application:**
    *   90/10 법칙: 일부 코드의 재사용률이 매우 높음 (보일러플레이트 코드)
    *   핵심 코드 일부를 효율적으로 관리
*   **Spring Application Context:**
    *   Spring API 구축 및 앱 실행에 필수
    *   `@Configuration`, `@ComponentScan` 등을 통해 Bean 관리
*   **의존성 (종속성):**
    *   다른 클래스의 속성/기능을 가져와 사용하는 것 (has-a 관계)
    *   상속 (is-a 관계)과 구분
    *   **의존성 주입 (Dependency Injection):** Spring Boot의 주요 장점
*   **IoC Container (Inversion of Control):**
    *   객체(Bean)를 관리하고 필요한 부분만 제공
    *   객체 생성 및 관리를 프레임워크에 위임
*   **Bean:**
    *   IoC Container에 생성된 객체
    *   **`@` 어노테이션**을 통해 생성 및 관리 (예: `@Component`, `@Service`, `@Repository`, `@Controller`)
*   **빌더 (Builder):**
    *   **종속성 관리:** 필요한 라이브러리(JAR 파일) 자동 다운로드 (예: Maven, Gradle)
    *   **빌드 자동화:** 컴파일, 테스트, 배포 프로세스 자동화
    *   **빌드 라이프사이클:** validate, compile, test, package, install, deploy 등

---

### 🌐 웹 관련 개념

*   **URI / URL:**
    *   **URI (Uniform Resource Identifier):** 자원을 식별하는 고유 식별자
    *   **URL (Uniform Resource Locator):** 자원의 위치 주소
    *   **URN (Uniform Resource Name):** 자원의 이름 부여 (예: `urn:isbn:0451450523`)
*   **Endpoint:**
    *   URL에서 서버 주소를 제외한 **자원의 경로**
    *   예: `https://github.com/potatostore/Obsidian.git` 에서 `/potatostore/Obsidian.git`
*   **REST API:**
    *   마이크로서비스 아키텍처에서 각 서비스 간의 데이터 상호작용 인터페이스
    *   이점: 트래픽 관리 용이, 서비스별 독립적 재배포 가능

---

### ⚙️ Spring Boot 구성 요소 (프로젝트 구조)

*   **.idea:** IDE 설정 파일
*   **.mvn:** Maven 관련 설정 파일 (Maven 사용 시)
*   **src:** 소스 코드 디렉토리
    *   **main:**
        *   **java:** Java 소스 코드 (핵심 로직)
        *   **resource:** 정적 리소스, 설정 파일 (DB, Web 변수 등)
*   **pom.xml:** Maven 프로젝트 설정 파일
    *   의존성(라이브러리) 버전, 아티팩트 ID 등 명시
    *   빌더(Maven)가 라이브러리를 다운로드하고 관리하는 방법 정의

---

### 🛠️ 도구

*   **Spring Initializr:** 프로젝트 생성 도구 (Compiler, Language, Model, 의존성 설정)
*   **HTTPie:** Spring Boot API 테스트 도구
    *   `pip install httpie` 로 설치
    *   Jackson 라이브러리를 이용한 마샬링/언마샬링 지원

---

### ✨ 주요 어노테이션 (예시)

*   **`@Configuration`**: 클래스를 Bean 설정 클래스로 지정
*   **`@ComponentScan`**: Bean을 찾을 패키지 지정
*   **`@Bean`**: 메소드 반환 객체를 Bean으로 등록
*   **`@Component`**: 일반적인 Bean 등록
*   **`@Service`**: 서비스 계층 Bean 등록
*   **`@Repository`**: 데이터 접근 계층 Bean 등록
*   **`@Controller`**: 웹 계층 컨트롤러 Bean 등록
