# W2

## 유비쿼터스 혁명
#### 도시 혁명
- 자급자족의 농촌 마을에서 복잡하고 문명화된 대규모의 도시사회로 변화하는 과정
- 시간과 거리를 줄이기 위해 도시에 모여 생활

#### 산업 혁명
- 철도, 전화를 통해 활동 반경 확대
- 토지, 노동에만 의존하던 공간에 에너지를 도입하여 공간이 갖는 생산력 확대

#### 인터넷 혁명(정보 통신 혁명)
- 정보 기술의 발달로 전자공간 등장
- 사회경제적 활동을 전자공간에서 수행 -> 실체적 공간으로 급성장

#### 유비쿼터스 개념 등장
- 마크와이저의 유비쿼터스 컴퓨팅 개념으로 전자 공간 중심의 정보화 패러다임에 또 다른 대전환

#### 공간 혁명(유비쿼터스 혁명)
- 기존 전자공간과 물리공간의 이분화에서 벗어나 이 둘을 유기적으로 합침 -> 유비쿼터스 공간으로 진화
- 물리 공간의 모든 대상을 연결시키는 통신 대상의 무한 확장 개념
- 기존 컴퓨터 중심적이었던 컴퓨팅 시스템에서 사람-사물-컴퓨터 간의 연계 주장(전자공간과 물리공간의 연계 의미)
- 각종 사물과 물리적 환경 전반(물리공간)에 컴퓨터가 적절히, 효과적으로 삽입되어 보이지 않으면서(투명성), 정보흐름이 이뤄저야 한다고 주장

## 전자공간과 물리공간의 연계 접근 방법
#### 전자공간과 물리공간의 연계에 대한 접근 시각
- 증강현실을 전자공간과 물리공간의 연계로 보는 시각
	- 휴대용 비디오 카메라를 매개체로 현실세계의 이미지를 해석하여 컴퓨터가 인식
- 유비쿼터스 컴퓨팅 개념에 입각한 시각
	- 물리공간 속 수많은 작은 컴퓨터를  통해 사물-사물, 사물-컴퓨터, 컴퓨터-사물 상호작용을 인식
	- 사물-컴퓨터-사람이 네트워크로 연결되어 소통

#### 연계 성립 조건
- 사물-컴퓨터-사람의 네트워크화 차원 : 물리공간에 심어진 작은 컴퓨터가 네트워크를 통해 연결되어야 함
- 물리적 속성의 인식 차원 : 물리공간에 심어진 작은 컴퓨터가 전자공간에서의 응용/서비스 수행에 필요한 물리적 속성을 포착 및 통식, 구동 가능
- 물리적 속성과 전자적 속성 간의 연계 차원 : 물리공간의 속성 상태는 전자적 속성과 상호작용 가능

-> *전자공간과 물리공간 간의 연계 : 네트워크 기반, 물리공간에 대한 상황인식 가능, 물리공간/전자공간 간 양방향 상호작용 가능*

#### 공간과학성
- 물리공간/전자공간의 한계 극복, 차세대 정보 서비스와 응용 개발에 중요한 방향 제시

#### 전자공간/물리공간의 연계에 대한 기본적 매커니즘
- 전자공간/물리공간 속성들이 사용자가 요구하는 서비스 제공 가능하도록 시스템적 상요작용하는 것이 핵심

#### 웹 현실화 기술을 활용한 전자공간과 물리공간의 연계
- 물리공간 3대 요소(사물,사람,컴퓨터)를 웹상에 표현하는 기술
- 사용자의 상황에 기초하여 물리적 속성에 관한 웹 정보 내용의 자동 생성 및 상호 관계성 유지 기술

# 3W

## 비즈니스 패러다임의 변화
- 많이 만들자(2차 산업혁명) -> 사용자 맞춤 제작
- 연결/제어 중심의 사물인터넷 -> Edge computing, AIoT / Physical AI 등을 활용한 지능의 고도화된 사물 인터넷

| 구분     | 기존 경쟁           | 신규경쟁                 |
| ------ | --------------- | -------------------- |
| 가전 제조  | 제조사 간 경쟁        | 유통 기업과 제조사 간 경쟁      |
| 의료 서비스 | 사후 치료 방식        | 사전 건강관리 방식(예방중심)     |
| 물류 산업  | 가격 경쟁 중심        | 소비자 맞춤형 배송 경쟁        |
| 제조 산업  | 대량 생산, 제조/판매 산업 | 맞춤형 적량/대량 생산, 서비스 사업 |
1. 냉장고를 무료로 제공하는 유통 기업 : 비즈니스 패러다임의 변화로 이전과 추구하는 냉장고의 모습이 달라질 수 있다(신선 보관 목적에서 신선식품 소비 목적으로 변환)
2. 사후 치료에서 사전 예방 중심의 의료 산업(IBM Watson의 현장검사진단, 컴퓨터 단층 촬용, 자기공명화상을 통한 특정 질병 발병 여부 판단 -> 오진률 감소)
3. 가격에서 맞춤배송 경쟁으로 변화하는 물류산업 : 소비자가 원하는 시간과 장소에 배송(아마존 프라임 나우, 프라임 에어)
4. <span style="color:rgb(0, 176, 80)">정보통신기술(ICT) 융합에 의해 선진화되는 제조업</span>
	- 제조공장을 다시 본국으로 회귀 : 기존의 인건비보다 신기술 보호와 같은 기타 요소들이 더 중요해짐
	- 제조의 서비스화 : 제품에 다양한 서비스를 결합하여 차별화 포인트를 부가시켜 제품을 고부가가치화
	- 제조과정의 지능화 : ICT기술을 제조기술에 융합하여 생산성 극대화
	- 제품 개인화 : 맞춤형 제품 생산
	- 제품 판매에서 서비스 판매로 사업 모델 변화
	-> 기존 소품종 대량 생산을 맞춤형 적량/대량 생산으로 패러다임이 변화

- 사람과 사람을 둘러싸고 있는 다양한 사물이 생성하는 데이터 간 연결을 통해 사회 전 분야에 변화와 혁신을 이끄는 사회

# 4W
## IoT

#### 정의
- 사물들이 단순히 인터넷을 통해 서로 연결되는 것을 넘어서 가치를 추가하여 그 가치를 사용자에게 제공하는 기술
- 모든 것들이 인터넷으로 연결되어 상호소통
- 사물인터넷 생태계 : S-P-N-D-Se 구조(service, platform, network, device, security)

#### 사물인터넷 활성화 요소
- 4대 활성화 요소 : 소형화, 저전력화, 저가격, 표준화를 통해 실현
- 소형화 : 미세전자제어기술, 나노 기술
- 저전력화 : 소형화된 소자는 적은 전력 소모로 저전력화 실현, 와이파이보다 저전력 블루투스 기술 채택
- 저가격 : 소형화 및 대량생산 기술의 발전으로 실현
- 표준화 : 표준화된 인터페이스를 통해 데이터 교환이 가능하도록 모듈화

#### 발전 단계
- 1단계(연결형) : 사물이 인터넷에 연결되어 주변 환경을 센싱하고 그 결과를 전송, 원격에서 사물이 제어되는 단계(센싱-수집-관리(분석)를 목적으로 구축된 실 세계에서 가상세계로의 연결 및 관리 기술)
- 2단계(지능형) : 사물이 센싱 및 전송한 데이터를 분석 및 예측하는 지능적 행위를 취할 수 있는 단계(1단계에서 지능이 추가되어 물리공간에서 가상공간으로의 지능적 감지, 유연 대응에 대한 기술)
- 3단계(자율형) : 사물간 분산 협업 지능을 기반으로 상호소통, 스스로 의사결정 및 자율적으로 제어하는 단계(2단계 기술 고도화 + 대응 자율화 기술을 통해 지속적인 물리공간-가상공간의 상호 작용 사이클 완성)

## AIoT
- AI와 IoT 기술을 결합
- 초연결(연결성) + 초지능(지능성) = 초융합영역으로 확장

#### 관련 기술
- 지능화를 위한 AI 기술(빅데이터, 머신러닝 등)
- 끊임없는 서비스 제공을 위한 유무선 통신 기술
- 보안 및 여러분야 서비스 제공을 위한 융합서비스와 솔루션

- IoT 디바이스 및 플랫폼 : 칩, 모듈, 게이트웨이, 디바이스 플랫폼, 오픈 H/W플랫폼
- AI 기술 및 하드웨어 : 빅데이터/머신러닝/딥러닝, AI칩
- 유무선 통신 기술 및 네트워크 장비/부품 : 와이파이, 비콘, 블루투스
- 보안 제품 및 솔루션 : 디바이스 보안, 통신/네트워크 보안
- 융합서비스/솔루션 : 스마트홈, 스마트시티, 탄소중립 등

#### 구현 방식
- 클라우드 지능을 활용하는 방식
	- 지능형 클라우드 플랫폼 활용 : 아마존 알렉사, 올리, 페퍼 등 글로벌 IT 기업의 클라우드 플랫폼이 제공하는 클라우드 서비스
	- 지능형 IoT 서비스 클라우드 플랫폼 활용 : 인공지능 가전, 대화형 비서 등 하드웨어 사물 제조사가 응용서비스 제공을 위해 구축한 서비스 클라우드 플랫폼에 인지, 분석기능 등을 추가하여 지능화된 IoT 서비스 제공
- 사물 지능화
	- 지능화 엔진 사물 탑재 : 학습 알고리즘 기반 지능화 엔진을 제품에 탑재하여 인지 및 사고 기능 보유 (네스트 서모스탯, 박스터 로봇)
	- 지능형 사물 플랫폼 및 인지 도구 활용 : 자율주행차와 같이 특화된 지능을 요구하는 사물에 탑재하기 위한 사물 플랫폼 활용 (퀄컴 드라이브, 데이터 플랫폼, 쿼크)

#### 플랫폼
- AI기반 스마트하우징 플랫폼 : 상황 대처에 최적화된 자동화 기능 수행
- 국제표준기반의 AIoT 플팻폼(NeoIDM)

#### 정책 및 지원 사례
- 지능형 IoT 기반 실내 공기질 관리 시스템 : 시내외 센서를 통해 공기질을 측정 후 자동 관리
- IoT 적용 VR/AR 재활치료 시스템 : 환자의 생체 데이터 수집 및 필요한 재활 서비스를 제공
- AIoT 기술 활용 다중이용시설 통합관제 서비스 : 다중이용시설의 각족 피해를 예방하기 위해 전기실, 발전기 등에 센서를 설치 후 측정, 이상징후 발생시 AI를 통해 파악 후 조치
- AI 반자율운전 신재생에너지 하베스팅 시스템 : 소수력 발전기 반자율운전 시스템
- 지능형 IoT 통합 콜드체인 서비스 : 콜드체인 인프라에 센서를 설치하여 모니터링 및 AI를 통해 사고 발생전 인지
- 중소장비 제조기업용 예측정비 서비스 : 제조장비에 센서 부착하여 데이터 수집하고, AI를 통해 징후 예보 -> 생산성 증가
- AIoT 로보틱 악기 등 기반 비대면 공연장
- 안전 분야 / 국민편의 제고 분야에 지원

	- 사람의 조작 없이 스스로 운행이 가능한 자동차
	- Electric / Connected / Self-driving
	- 자동화 단계 : 미국 도로교통안전국
		0. 수동 조작
		1. 특정 기능의 자동화 단계인 선택적 능동제어 단계(크루즈)
		2. 기존 자율주행 기술들이 통합된 능동제어 단계(오토파일럿)
		3. 특정 상황에서만 운전자의 개입이 필요한 제한적 자율주행단계(구글 자율주행차)
		4. 운전자의 개입이 필요 없는 완전자율주행 단계

	- 미국 자동차기술 학회(SAE)의 자동화 단계
		0. 비자동 : 운전자가 모든 조작 제어
		1. 운전자 지원 : 조향 지원시스템 및 가속/감속 지원시스템 실행, 사람이 자동차의 동적 주행에 대한 모든 기능 수행
		2. 부분 자동화 : 조향 지원시스템 및 가속/감속 지원시스템 실행, 주행환경의 모니터링은 사람, 안전운전 책임도 운전자가 부담
		3. 조건부 자동화 : 시스템이 모든 운전 조작 제어, 운전자가 적절히 자동차 제어
		4. 고도 자동화 : 모두 시스템이 수행, 시스템이 전적으로 항상 제어하지 않음
		5. 완전 자동화 : 모든걸 시스템이 담당
	- 반자율주행 기능(첨단운전자보조시스템, 부분자동화 수준) : 전방 충돌방지 보조(FCA), 차로 이탈방지 보조(LKA), 후 측방 충돌방지 보조(BCA), 안전하차 보조(SEA), 지능형 속도 제한 보조(ISLA), 운전자 주의 경고(DAW), 후 측방 모니터(BVM), 스마트 크루즈 컨트롤(SCC), 원격 스마트 주차 보조(RSPA)

## 반도체와 IoT

#### 반도체
- 저항으로 물질을 구분하고, 전류가 잘 통하면 도체, 잘 안통하면 절연체인데, 외부의 인가된 전원에 따라 이중적인 특성을 갖는 물질을 반도체라고 함
- 4차 산업시대가 도래하며 나노기술, 정보통신기술, 생명공학기술이 융합된 IoT 반도체 산업이 주를 이룰 것으로 전망
- 변환 칩 : 외부에서 수집되는 데이터(온도, 압력, 소리 등)을 디지털 데이터로 변환
- 통신 칩 : cpu연산 처리 후, 인터넷 접속 기능 수행
- 시스템 반도체 : 메모리를 제외한 나머지 반도체
- <span style="color:rgb(146, 208, 80)">반도체 웨이퍼 : 칩을 만드는 바탕판, 반도체 집적회로를 만드는 중요한 재료</span> 

#### 메모리/비메모리 반도체
- 메모리 반도체(RAM / ROM)
	- ROM : 읽기전용 기억장치, 비휘발성 메모리
		- PROM : 1회에 한해 기록, 내용 수정 불가
		- EPROM : 수시로 삭제 및 수정 가능(자외선을 통해)
		- EEPROM : 전기적 신호를 보내 기억된 데이터 삭제(플래시 메모리)
	- RAM : 휘발성 기억 장치, 전원 공급이 있어야만 정보 유지 가능
		- SRAM : 빠른 속도, 캐시메모리
		- DRAM
- 비메모리 반도체(시스템 반도체) : 정보 저장 x, 논리/연산/제어 기능 등을 수행

#### 뉴로모픽 반도체
- 인간의 뇌, 신경세포의 구조와 특성을 모방하여 효율성을 높인 AI형 반도체(IoT, AI, 빅데이터, 자율주행차 등 핵심 기술이 위치)
- 기존 폰노이만의 구조(cpu + ram + 입출력장치)에서 벗어나 하나의 반도체가 모두 처리하기에 속도가 빠름(병목 현상 x)
1. 병렬 연산 구조로 정보처리 속도가 빠름
2. 두뇌의 핵심 기능인 패턴인식 가능
3. 기존 반도체에 비해 전력 소모량이 크게 감소

#### WiFi

#### Bluetooth
- 10m 안팎의 초단거리에서 데이터,음성 등 저전력 무선연결에 사용하는 무선기술
- 주파수 호핑 : 다수 채널을 특정 패턴에 따라 빠르게 이동시키면서 패킷을 조금씩 전송하는 기술로 전파 간섭 방지
- Master - Slave로 구성

#### RFID
- 무선인식
- 태그와 안테나를 통해 서로 인식
- 능동형 : 전원 필요, 리더기와의 인식 거리를 멀리할 수 있고, 필요전력을 줄임 / 전원공급장치에 따른 작동시간 제한 및 고가
- 수동형 : 전원공급 없이 리더기 전자기장으로 작동, 매우 가볍고, 반영구적 사용 / 인식 거리가 짧고, 전력소모가 큼
- 저주파수 태그 : 저렴한 태그, 짧은 판독 거리, 소량의 데이터 저장
- 고주파수 태그 : 판독거리가 길어 움직이는 물체나 빠르게 인식되는 물체에 적합

- <span style="color:rgb(0, 176, 80)"> 동작 절차</span>
	1. 활용 목적에 맞는 대상에 부착
	2. 리더에 안테나를 통 태그를 향해 신호 송출
	3. 태그의 데이터를 리더에게 송출
	4. 안테나가 수신한 데이터를 디지털 신호로 변조하여 리더로 전달
	5. 리더가 데이터를 해독하여 호스트로 전달

#### NFC
- 근거리 무선통신, 비접촉식 양방향 근접 통신 기술
- 상황에 따라 리더와 태그의 역할을 변경
- 기기에 NFC기능이 탑제된 스마트폰을 태그시 연결

- P2P 모드 : 두 대의 NFC 디바이스가 상호 데이터 송수신
- 리더/라이터 모드 : RFID 태그 인식을 위한 리더 필요, 태그 인식을 위한 전력 필요
- 카드 애뮬레이션 모드 : 전력공급 불필요, RFID 태그를 카드처럼 사용

#### ZigBee
- 홈 오토메이션을 편리하게 하기 위한 목적으로 개발된 기술
- 저전력 및 저가

- 단말장치 : 각종 센서에 연결되어 데이터 전송
- 코디네이터 장치 : 정보 수집 및 장치 제어
- 라우터 장치 : 여러 장치 간 데이터 전달

## 유비쿼터스 사회의 전체적인 시스템

### 유비쿼터스 사회 시스템(3단계의 비즈니스 계층)
- 유비쿼터스 - 사회 시스템 응용 프로그램
- 유비쿼터스 - 사회시스템 플랫폼
- 유비쿼터스 - 네트워크

#### 시스템 응용 프로그램 
- 인간계
	- U-건강안심 시스템 : 언제 어디서나 누구든지 자신의 건강상태를 파악할 수 있게 만드는 시스템
	- U-자동차 네트워크 시스템 : 운전자 편의 서비스(신시간 교통 정보, 예상 소요시간 정보 등), 도로관리자 편의 서비스(실시간 도로 교통상황 파악 날씨예측 등)
	- U-교육학습 시스템 : 언제, 어디서나 다양한 학습 프로그램을 이요하는 시스템
- 환경계
	- 국민 생활환경 안정성, 사회자본 효율성 제고를 도모하는 사회 시스템
	- U-식품 이력/안정성 확인 시스템 : 식품의 포장 위에 식품 속성 정보를 기록한 U-칩 부착
	- U-도로환경 센싱 시스템 : 자동차가 운행 중에 기본적인 위치, 기상 등의 정보를 U-단말기로 제공하는 시스템

#### 사회시스템 플랫폼
- 유비쿼터스 단말기 : U-네트워크와 U-사회 시스템 응용 프로그램을 연결하는 인터페이스, 다양한 응용 시스템에 공통적으로 사용할 수 있는 기기
- 유비쿼터스 칩 : 물리적인 사물에 식별 정보를 담은 RFID 및 외부 상황을 감지하는 센서 의미
- 유비쿼터스 서비스 유통센터 : 다양한 U-사회시스템 응용 프로그램에 의해 제공되는 서비스의 수요/공급을 연결해주는 가상 서비스 수급 조정 창구

#### 네트워크
- PC뿐만 아니라 다양한 종류의 단말기가 네트워크에 항상 접속, 누구든지 시간/장소 제약 없이 네트워크 활용하여 다양한 서비스를 제공받을 수 있는 환경 또는 이를 가능하게 하는 기술 -> 광대역 통신 인프라, 무한대의 IP주소부여 기술 등을 통한 상시 접속
- 차세대 네트워크 기술(NGN)을 이용하여 언제 어디서나 원하는 서비스를 받을 수 있는 5Any 동작 요구하는 다양한 종류의 프로그램 및 서비스를 지원하기 위한 네트워크
- 기본 환경 : 사물 to 사물을 지원하기 위해 제공되는 네트워크 환경으로 고정사물, 이동 사물, 논리적 사물(서버상의 콘텐츠 같은 객체)를 유무선 인터페이스를 통해 NGN에 연결 -> 모든 것을 연결(Barrier Free Interface를 통해 Non-PC 중심으로 24시간 현실 생활 지원)

	- 두 개 이상의 기지국 혼용 이종 네트워크 HetNet에 기초한 무선 접속기술 활용으로 용량, 지연, 에너지 효율 증대 필요

- 소프트웨어 정의 네트워크 기술(SDN) : 네트워크 장비의 제어 부분을 데이터 전송 부분과 분리, 산업표준 제어 프로토콜 사용, 네트워크 경로 설정과 제어

3. 클라이언트는 HTTP표준에 따라 형식화된 요청 메시지 발송
4. 서버는 요청을 읽고 해석 -> 응답 메시지 클라이언트에게 전송

#### REST 개요
- IoT 기기의 상태 전송이란 뜻으로, 인터넷 정보를 조직하고 전송하는 규칙의 조합
- 리소스라는 이름으로 인터넷상의 문서, 이미지, 서비스와 같은 정보 지칭
- 클라이언트-서버 네트워크 환경에서 리소스 CRUD 처리 지원

#### CoAP
- 인터넷에서 센서 노드와 같이 제한된 컴퓨팅 성능을 갖는 디바이스 통신 실현을 위해 IETF의 CoRE워킹 그룹에서 만든 응용 계층 표준 프로토콜
- TCP, HTTP와 같은 무거운 통신 프로토콜 사용 불가 -> 웹 서비스에서 사용 가능한 가벼운 프로토콜 개발 목적으로 설계된 기술

#### MQTT 
- 제한된 컴퓨팅 성능과 빈약한 네트워크 연결 환경에서의 동작을 고려하여 설계된 대용량 메시지 전달 프로토콜
- IoT표준 프로토콜
- Publisher / Subscriber 관계에서 Broker라는 중계 서버를 기반으로 IoT기기 간의 데이터 전달
- 전달되는 메시지에는 손실복구 기능 지원

#### XMPP 
- 인스턴트 메신저용으로 IETF에서 제정한 국제 표준 프로토콜
- publisher / subscriber 구조를 바탕으로 확장성 있는 XML기반 실시간 메시지 교환 가능

## IoT 보안기술

#### 보안 위협 사례
- 정보통신망 침입
- 디도스 공격
- 악성 프로그램 유포
- 데이터 침해
- IoT 이용 살상 행위
- 기간시설위협

- 디바이스 및 센서 : 비인가 접근, 복제 공격 등
- 네트워크 : 정보 유출, 서비스 거부, 인증 방해 등
- 서비스 : 프라이버시 침해, 비인가 앱 및 사용자 접근 등

#### 보안 공격 유형 
- 비인가 접근 : 특정 장치나 지원, 서비스에 비인가 접근 시도 후 내용을 조작하거나 물리적 손상을 입힘, 원격 제어 기기들은 더 취약 -> 역할 기반 접근 제어, 속성 기반 접근 제어 등의 접근 제어기법 사용
- 정보유출 : 기존 정보통신 서비스의 정보 유출과 거의 동일한 형태
- 유무선 통신 공간에서 도청, 스니핑, 비인가 접근에 의한 유출
- 데이터 위변조 : 비인가 장치나 센서를 인가된 것처럼 가장하여 데이터 전송 및 가로채기
- 프라이버시 침해 : 개인정보 유출
- 서비스 거부 : 서비스 제공으 루이한 연결 요청 시 대량의 접속신호를 한꺼번에 발생시켜 지연 마비

#### 디바이스 인증 / 식별 기술
- 악의적 디바이스로 인한 오동작, 의도하지 않은 기능 수행 방지를 위해 디바이스에 대한 인증 및 식별 필요
- 기술
	- 인증서 사용
	- ID/패스워드 사용
	- 토큰 사용
	- SIM과 같은 식별 장치 사용
	- 유일한 식별자 사용

#### 디바이스 경량 암호 / 보안 프로토콜 기술
- IoT 디바이스 : 직렬 주변 장치 인터페이스 수준의 시리얼 통신 수행 디바이스
- 다양한 디바이스에서 센싱 된 데이터의 기밀성, 무결성 등 기본적인 보안 요구 사항 구현을 위해 해시값, 암/복호화 등을 생성하는 암호 알고리즘 필요
- 기술
	- 데이터 암호화 규격(DES) : 대칭키 암호화 방식(암호화 및 복호화 시 같은 대칭키 사용) -> 너무  짧아 해독 가능
	- AES : DES 함호화 기술 문제 해결을 위해 개발된 암호화 방식 -> 대칭키를 사용하지만 키의 길이가 길어짐

#### CoAP, LWM2M 보안
- CoAP : 제한적 자원을 가진 디바이스와 HTTP 기반의 응용 서비스 간 통신에 적합한 프로토콜로 REST 통신 용이, DTLS를 통해 보안
- LWM2M : CoAP에 기반하여 클라이언트-서버 간 상호 인증을 통해 서비스 제공, 통신 메시지에 대해 기밀성/무결성 제공 가능

#### MQTT 보안
- MQTT 프로토콜에 사용되는 보안 방식
- MQTT 프로토콜 : IoT 서비스에 특화된 프로토콜, 이기종 기기 및 서비스 간 통신에 사용
- 인증 메커니즘 구체화 및 자원 접근 제어등 통신 메시지 무결성과 프라이버시 보호 기법 정의 필요 -> 프로토콜에 암호 알고리즘 및 보안 프로토콜 미정의, 서비스 제공자에게만 보안 책임 부여

#### 접근 제어 / 권한 제어 기술
- 서비스 영역에서의 대표적 보안 이슈는 서비스 접근 제어
- 접근 제어 주체 및 방법에 따른 구분
	- 임의적 접근 제어 : 서비스 제공자가 서비스 사용자에 대한 권한을 직접 부여
	- 강제적 접근 제어 : 임의적 접근 제어 기법의 보안 문제 보완을 위해 제공자 대신 공통 플랫폼을 통해 IoT 관리자와 같은 최고 권한자의 서비스 사용자에 대한 권한 설정 방식
	- 역할 기반 접근 제어 : 비임의적 접근 제어로 분류 -> 사용자에게 역할 부여 및 서비스 접근 권한 부여하는 방식으로 단순화 가능
	- 속성 기반 접근 제어 : 사용자 인증 수행, 사용자의 권한이 아닌 속성에 따라 접근 제어

#### Open API 기술
- IoT 서비스에서 다른 서비스 자원 접근을 위해 사용
- 사용 권한 확인용 사용자 인증 메커니즘을 통해 다른 IoT서비스에서 서비스 접근 및 사용 가능
- OpenID 방식 : ID를 제공하는 3자 서비스를 통해 OpenID를 인증받고 공개 API를 사용하는 기법, 악의적인 RP에 의한 피싱 공격 가능
- SAML : XML기반, SAML 요청을 통해 인증과 인가 수행
-> 위 두 방식은 단일 인가를 목표로 제안
- OAuth 1.0 인증 방식 : 공개 API에 대한 인가를 목표로 개발

#### 서비스 보안 기술
- IoT 사용자의 편의성 향상을 위해 웹 서비스 기반으로 개발
- 웹 서비스 개발 사용 표준: WSDL, UDDI, SOAP
- SOAP : XML 기반의 RPC 프로토콜로 실제 메시지 송수신의 핵심 프로토콜, OASIS 제안(WS-Security, SAML, XACML), W3C 제안(XML Signature)

#### ID 관리 기술
- ID 관리 시 개인 정보 누출, 주민번호 도용 등 문제 발생 -> URL 기반 인증 수행 방식인 OpenID는 ID관리를 위한 대체 기술로 주목

## 블록체인
- IoT : 사람,사물,공간,데이터 등 모든 것이 인터넷으로 서로 연결되어 정보가 생성/수집/공유/활용되는 초연결 인터넷
- 블록체인 : 거래 다상 데이터를 Block이라는 상자에 저장, P2P 기반의 네트워크에 전파 후 블록에 참여 노드들의 합의를 거쳐 체인 형태의 연결고리에 연결되는 분산 데이터 원장관리 기술
- IoT, 블록체인의 융합 -> 사물인터넷 플랫폼으로부터 얻는 데이터를 처리하는 융합 서비스

#### IoT - 블록체인 관계
- 비트코인 : 가상 자산으로, 기록의 신뢰성을 공인된 제 3자가 없어도 확보 가능
- 사물인터넷 : 블록체인 기술을 이용하여 IoT 데이터의 무결성 유지와 거래 활성화
- 중고차 판매 : 생태계의 모든 것이 담긴 기록 즉 소유권, 금융, 등록, 보험과 서비스 거래 등을 모두 추적 가능

#### 장/단점
- 장점 
	- 제 3자의 개입 불필요
	- 해킹, 조작에 안전
	- 거래 내역이 투명하게 보관
	- 경제 비용 절감
- 단점
	- P2P방식의 분산 시스템으로 인한 속도 저하
	- 신속한 대응 불가 및 기술 이슈
	- 안정적이지 못하고, 비싸고, 복잡하고, 규제되지 않았으며, 완전한 신뢰 불가

#### 특징
- 거래 비용절감
- 안전하고, 편리한 데이터 활용
- 사물 인터넷 기기 간 자율협업 지원을 통해 4차 산업혁명 구현을 지원

#### 발전과정
- 디지털 화폐 -> 스마트 계약이나 분산 앱을 통해 다양한 비즈니스에 적용 단계 -> 사회 전반에 블록체인 기술 적용

#### 매커니즘
- 블록 생성부터 검증, 분산 합의를 거쳐 신뢰성 있는 데이터블록이 블록체인에 연결되는 전체 프로세스
1. 거래(블록체인 네트워크에서 가상자산 이동에 따른 약속된 전송서식) 정의 및 거래 생성, 유효성 검증
2. 거래가 블록에 묶이기 전 임시로 저장되는 곳(메모리 풀)에 등록 
3. 블록(유효한 거래 정보의 묶음) 형태 구성 및 해시작업을 통해 유효성 검증
4. 해시값 확인 및 유효성 상호 검증
5. 블록체인에 등록

#### 블록
- 블록해시 : 블록의 식별자 역할을 수행하는 블록의 이름정보
- 블록 헤더 : 80byte크기의 정보들
- 블록 바디 : 코인베이스(트랜잭션과 여러 거래 정보로 구성), 트랜잭션 카운트(거래 개수), 코인베이스 트랜잭션(본 블록을 마이닝한 마이너의 수입), 트랜잭션(10분동안 수집한 거래 정보)

#### 머클루트 및 블록해시 산정
- 머클트리의 최상위에 위치하는 해시값을 머클루트라고 함
- 블록해시 : 블록의 이름정보, SHA256으로 변환한 결과값
- 채굴 : 블록해시값을 식별자로 가지는 유효한 블록을 만들어내는 작업 증명을 의미

- UTXO : 미사용 트랜잭션 출력값, 소비되지 않은 출력

#### 관련 기술
- 블록 : 논리적인 데이터 단위
- P2P 네트워크 : 개인 컴퓨터들이 네트워크로 연결되어 서로 정보를 공유하는것
- 암호화 해시함수 : 해시값으로 입력값과 관계를 찾기 어려운 성질을 가지는 경우 의미
- 작업증명
- 스마트계약 : 특정계약을 스스토 검증하기 위한 컴퓨터 프로토콜
- 블록체인 : 블록들이 생성될때마다 기존 블록들의 연결에 새롭게 추가 연결
- 블록체인 네트워크 : 전체 블록체인을 노드가 복제하여 자기 노드에 저장
- 해시함수 : 데이터를 고정된 길이의 문자, 숫자로 조합된 해시값으로 변환하는 알고리즘, 같은 입력값에 대해 항상 같은 출력값 -> 익명 거래 수행, 무결성검증, 채굴
- 전자서명 : 송신자의 신원을 증명하고, 변조되지 않았는지 확인
- 공개키 암호화 : 사전에 개인키를 나눠 가지지 않은 사용자들이 안전하게 통신할 수 있도록 해주는 방식
	- 공개키 : 누구나 알 수 있음
	- 개인키 : 소유자만 알 수 있음
- 디지털 키 : 네트워크상이 아닌 지갑속에 생성해서 저장(공개 키 + 개인키)
- private Keys : 개인키를 통해 공개키 생성, 공개키로 비트코인 주소를 생성
- public Keys : 누구에게나 공개된 키
- 주소 : 외부로 공개되는 주소, 코인을 주고 받기 위해 사용
- 블록체인 지갑 : 개인키, 공개키 저장 장소
	- 핫월렛 : 온라인 상태, 신시간 거래 정보 교환 가능
	- 콜드월렛 : 인터넷 연결 x, 

#### 핵심 기술
- 중심 서버를 두지 않고 클라이언트간 통신하는 P2P 네트워크
- 분산 노드간 합의에 의해 거래가 수행되는 시스템
- 거래정보를 기록한 원장을 P2P네트워크에 분산 관리
- 장부에 기록되기 위해 유효한 서명이 필요하고, 서명에 필요한 디저털 키 구조

#### P2P 네트워크 
- 동등한 관계의 컴퓨터 그룹이 직접 통신하는, 중심이 없는 네트워크 
- 부하 분산
- 노드들이 서버, 클라이언트 역할을 동시 수행

#### BIoT
- 보안 안정성을 높이고, 누구나 투명하게 거래
- 빅브라더 문제를 해결하기 위한 분산형 블록체인 등장

## 빅데이터 
- 기존 기술로 데이터를 수집/저장/관리/분석하는 것이 어려울 정도로 방대한 규모의 데이터를 의미
- 3V 속성에 최근 2V 추가

#### 3V 
- Volume(크기) : 단순 저장되는 물리적 데이터 양을 나타내는 빅데이터의 가장 기본적 특징
- Velocity(속도) : 데이터의 고도화된 실시간 처리를 의미
- Variety(다양성) : 다양한 형태의 데이터를 포함하는 의미

#### 2V
- Veracity(정확성) : 데이터의 증가에 따른 엉터리 데이터 증가 -> 정확성 요구
- Variability(가변성) : 데이터가 맥락에 따라 의미가 달라짐
- Visualization(시각화) : 정보 가공 시 가공 정보에 대한 이해 용이를 위한 시각화

#### 빅데이터 다차원 분석
- 다차원 분석으로 위치기반 데이터와 소셜 미디어에서 생산되는 사용자 활동 데이터를 새로운 차원의 정보로 가공

#### 빅데이터 분석 알고리즘
- 지식발견(KDD) : 빅데이터 분석 사례로서 데이터에서 가치 있는 정보를 찾아 내는 일련의 행위를 의미
- 연관규칙학습 : 다른 성격과 일정한 규칙을 갖는 둘 이상의 데이터군을 찾아내는 방법
- 분류 : 특정 규칙 및 특성 기준으로 구분된 데이터군을 바탕으로 새롭게 추가될 데이터군을 찾아내는 방법
- 군집화 : 하나의 큰 데이터군을 통계적 기법을 활용하여 비슷한 특성을 지는 여러 개의 작은 묶음으로 만드는 방법
- 회귀분석 : 어떤 현상을 구성하는 종속변수 값의 변화가 하나 이상의 독립변수 값을 어떻게 변화시키는지, 찾아내는 방법
-> 하둡, R, 분산병렬처리기술, 클라우드 컴퓨팅, 빅데이터 플랫폼(빅데이터 기술의 집합체 및 관련 기술들이 준비된 환경) 등에서 활용


## 클라우드 서비스 
- 사진, 문서, 영상 등 다양한 형태의 콘텐츠를 클라우드에 저장 후 인터넷 접속을 통해 언제 어디서나 이용할 수 있는 서비스

#### 주요특성
- 인터넷 상에 저장된 자료들을 간단 조작
- 별도 프로그램 설치 없이 응용 프로그램을 통해 원하는 작업 수행 가능
- 여러 사람들이 동시에 파일을 공유하며 작업 가능
- 서버 사용량 만큼의 비용 지불

## 클라우트 컴퓨팅
- 사용자가 사용자 인터페이스 기능을 제외한 모든 컴퓨팅 자원이 인터넷 클라우드에 위치함을 의미하는 인터넷 기반 컴퓨팅

#### 특징 
- 클라우드에 접속하여 문서 저장 및 작업
- 저장매체(USB) 불필요
- 편리함 제공
- 사용한 만큼의 비용 지불
- 상황에 맞는 시스템 구축 및 운영
- 비용 절감
-> 서버 장애시 사용 못하는 단점 존재

#### 종류
- 폐쇄형 클라우드 : 특정한 기업 내부 구성원에게만 제공되는 서비스
- 공개형 클라우드 : 일반인에게 공개되는 개방형 서비스
- 혼합형 클라우드 : 특정 업무는 폐쇄형 클라우드 방식을 이용하고 기타 업무는 공개형 클라우드 방식을 함께 이용

#### 핵심 기술
- 분산 처리 기술 
	- 분산 컴퓨팅 : 여러 대의 컴퓨터에 나눠 작업 후 결과를 다시 모으는 방식
	- 가상화 : 서버는 한 대지만, 여러 대로 나눠 동시에 여러 작업 처리
	- 시스템 관리 : 가상 컴퓨팅 환경 준비
	- 서비스 플랫폼 : 클라우드 컴퓨팅 인프라에 인터페이스 제공

#### 클라우드 컴퓨팅 서비스
- IaaS : 클라우드 컴퓨팅 관련 HW 서비스 제공
- PaaS : 클라우드 컴퓨팅 관련 HW, SW 및 응용 프로그램 제공
- SaaS : 사용자가 필요로 하는 기능만 골라 사용 후 비용 지불

## 엣지 컴퓨팅

#### 사물 인터넷 지능화 기술
- 사물 스스로 동작하고, 지연 시간이 거의 없는 실시간 동작이 필요
- 디바이스들은 인터넷을 통해 플랫폼에 직접 연결 및 관리
- 인프라에 데이터 분석 기능 적용
- IBM Edgent : 사물이 직접 데이터를 분석할 수 있는 기능 보유 및 활용을 통해 이상 징후나 중요 사건 감지 시에만 서버에 데이터 전송토록 개발

#### 엣지 컴퓨팅 시스템 및 자원관리 필요성
- 시스코 예측 : 사물인터넷 디바이스 급증이 IP트래픽에 영향을 줄 것으로 예상
- 엣지 컴퓨팅 시스템 : 서버로 데이터를 바로 보내지 않고, 인접한 엣지 노드에 데이터를 분산하여 분석 및 처리

#### 엣지 컴퓨팅 시스템
- 클라우드 컴퓨팅을 통해 데이터 중앙집중화로 컴퓨터 자원을 갖지 않아도 고수준 서비스 활용 가능
- 반면 엣지 컴퓨팅은 엣지 노드에서 처리하는 기술 -> 사물인터넷처럼 지연시간이 짧아야 하는 서비스 특성상 클라우드 보다 엣지 컴퓨팅이 유리

#### 엣지 컴퓨팅 종류
- Fog Computing : 클라우드 - 포그노드 - 중앙서버 3계층
- Multi-access edge computing : 엣지 노드로 무선 기지국을 주료 고려
- Cloudlet : 오픈스택 기반의 클라우드 컴퓨팅 기술을 기반으로 엣지 컴퓨팅 구현

#### 엣지 컴퓨팅 구성
- 엣지 컴퓨팅 칩 : 엣지 노드 단에서 데이터 처리 가속화 기능
- IoT 플랫폼
- Edge 단말 : 라우터와 데이터 분석 기능 탑재 게이트웨이 등으로 구성

#### 엣지 컴퓨팅 프로젝트
- OPNFV : 글로벌 통신 사업자로 구성된 단체
- StarlingX : 컴퓨팅 독립 프로젝트
- Akraino Edge Stack : 저지연 기능 제공하는 엣지 플랫폼 구현 목표

#### 산업용 IoT 엣지 컴퓨팅
- 보안 강화 및 기술 융합
- 생산 설비를 센서와 IoT에 연결하여 데이터 수집
- 실시간 제어를 통해 제품 불량률 감소 등
- 신뢰성과 실시간성을 요구하는 분야에서 기술 적용
- EdgeX : 벤더에 중립적인 오픈소스 프로젝트

- 포그 컴퓨팅 : 지능형 분산 컴퓨팅 제안, 소비 대역폭 축소 및 접속 회수 축소
