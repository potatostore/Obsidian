응용 계층 하부에 있는 계층들의 기본 역할은 신뢰성 있는 데이터 전송을 보장한다는 것이다.



**클라이언트-서버 모델**

인터넷 환경에서 응용 계층의 연결은 세션 계층에서 설명한 방식을 그대로 사용한다.

하나의 서버 프로그램이 다수의 클라이언트에 응용 서비스를 제공하는 클라이언트-서버 모델은

인터넷 응용 환경에서 보편화된 연결 설정 방식이다.

이때 클라이언트 포트와 서버 포트를 연결해 통신을 하고

네트워크 연결은 전송 계층의 포트 연결로 구분된다.



**연결형 / 비연결형 서비스**

클라이언트-서버 모델을 구현할 때 고려할 사항은 연결형인지 비연결형인지이다.

비연결형 서비스는 UDP를 사용한 빠른 데이터 전송을 지원하지만 신뢰성은 떨어진다.

반대로 연결형 서비스는 TCP를 사용해 신뢰성있는 연결을 지원하지만 3-way handshake나 헤더의 내용이 UDP보다

무겁기 때문에 속도는 떨어진다.



**상태 정보**

연결형 서비스는 클라이언트와 서버 사이의 연결로 인해 둘 사이의 현재 상태라는 개념이 존재한다.

일반적으로 가장 단순한 상태 정보는 연결 설정과 관련된 정보이다.

만약 통신 프로세스의 상태 정보에 오류가 있거나 잃어버리는 경우 다운되기 직전 상태로 복구해야 하는

문제가 있다.



클라이언트에 원격 파일 서비스를 제공하는 파일 서버(유튜브)는 **비상태** 서비스의 대표적인 예시이다.

클라이언트가 서버에 파일 서비스를 요청하면 서버가 결과를 응답하는 절차가 하나의 단위로 완결된다.

즉, 연결 상태가 존재하지 않는다는 것이다.(요청-응답이 하나로 묶이기 때문)



**동시성 제어**

임의의 동작이 외형상 동시에 진행되는 것처럼 보이는 것을 동시성이라 한다.

앞서 설명한 클라이언트의 요청이 올 때마다 서버를 복사해서 포트를 할당하는 단일 세션 연결 방식에서

요청이 올 때마다 복사해서 여러 클라이언트의 요청을 받는 것 또한 동시성 제어의 예시이다.
