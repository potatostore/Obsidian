OSI 7계층 모델은 TCP/IP 기반으로 구현되고

운영체제의 일부로 구현되는 계층 4 프로토콜은 TCP와 UDP를 사용한다.



이때 상위 세 계층인 세션, 표현, 응용 계층은 사용자 프로그램을 구현되는데,

현대의 네트워크 체제에서는 세 계층의 기능을 하나의 프로그램으로 묶어 구현한다.

**네트워크 응용 프로그램, TCP/UDP, IP, 이더넷**

위의 4계층으로 표현된다.



### **세션 계층**



세션 계층은 **세션 연결의 설정**과 **해제**, **세션 메시지 전송**에 관한 기능을 수행한다.

**하나의 응용 프로세스**는 **동시에 여러 세션 프로세스와 세션 연결을 생성**할 수 있으며,

세션 연결이 설정되면 양쪽 프로세스가 세션 메시지를 주고 받는다.

TCP와 마찬가지로 세션 사용을 종료하려면 세션 연결 해제 과정을 거쳐야 한다.



**동기 문제 처리**

세션 계층에서 제공하는 가장 중요한 기능은 동기 문제를 처리하는 것이다.

동기 기능은 통신 양단에서(Hop by Hop)에서 서로 동의하는 논리적 공통 처리 지점을 의미하는

동기점을 지정하기 위해 사용한다.

주로 메시지 전송 과정에서 발생하는 오류를 복구하기 위해 사용한다.

따라서 동기점은 동기점이 설정된 시점 이전까지는 통신 양단이 서로 완벽하게 처리했음을 의미한다.

오류가 발생하는 경우, 동기점 이전 과정은 복구하지 않아도 된다.



**토큰**

세션 계층은 두 응용 프로세스의 대화를 관리하기 위해 토큰이라는 특수 메시지를 사용한다.

토큰을 보유한다는 것은 해당 토큰에 부여된 특정한 권리를 배타적으로 소유한다는 의미이다.



**-토큰의 종류**



**-데이터 토큰**

데이터를 전송할 수 있는 권리를 제공한다.

따라서 세션 사용자가 데이터를 전송하기 위해서는 반드시 데이터 토큰을 먼저 획득해야 한다.

**데이터 토큰은 하나만 존재해야 하는 것이 아니며 하나만 존재하는 경우 반이중 전송을 지원하게 할 수 있다.**



**-해제 토큰**

통신 양단 간의 연결 해제 과정을 제어하기 위해 사용한다.

임의의 사용자가 연결을 해제하려면 해제 토큰을 획득해야 한다.



**-동기 토큰**

세션 연결을 사용하는 과정에서 동기 처리가 필요한 지점에 사용한다.



**토큰과 동기점**

대용량 파일을 전송할 때는 통째로 전송하는 것보다 작은 단위로 나눠 전송하는 것이 오류에 대처하기 쉽다.

오류가 발생하는 경우 다시 전송을 해야 하는데 만약 통째로 전송을 하면 처음부터 다시 보내야 하기 때문이다.

이때 큰 파일을 작은 단위로 나누는 과정을 **동기점을 부여한다고 한다.** 중간에 동기점을 부여해,

송수신 프로세스가 해당 위치까지는 데이터 전송이 완료되었다는 것에 합의할 수 있다.

따라서 전송 과정에서 오류가 발생해도 가장 가까운 지점에 설정한 동기점 이후의 데이터만 복구하면 된다.



동기점을 부여할 때 사용하는 동기 토큰은 2가지가 존대한다. 

1. 특정 대화 단위를 구분하는 **주동기 토큰**
2. 대화 단위 내에서 다시 작은 부분으로 나누어 처리하는 **부동기 토큰**



**재동기 기능**

세션 계층의 상위 계층에는 적절한 구간으로 나뉜 지점에 동기점을 부여하고, 오류가 발생하면 

해당 지점으로 돌아가 복구하는 기능이 있어야 한다.

동기점을 이용해 복구하는 과정을 **재동기**라 한다.



**주동기점과 부동기점**

동기점은 주동기점과 부동기점으로 나뉘고 전송 단위는 대화로 정의한다.

이 둘을 구분하는 이유는 부동기점에서는 복구에 필요한 백업 정보의 처리량을 주동기점보다 적게 해

복구에 소요되는 부담을 최소화하기 위함이다.

하지만 부동기점에 저장된 백업 정보로는 오류 복구를 완벽하게 할 수 없을 가능성이 있기에

이때는 앞의 부동기점에서 다시 데이터를 불러와 복구한다. **부동기점의 복구 절차가 진행되어도** 

**복구가 완전하게 이루어지지 않을 수 있다.**



재동기 처리는 직전의 부동기점으로 이동하는 과정을 반복하며 진행된다.

**이때 어떠한 경우에도 주동기점의 경계를 넘어 이동하지 않는다.**

주동기점이 부여되었다는 것은 해당 지점까지 데이터 전송이 완벽하게 이루어졌다는 뜻이기 때문이다.



**액티비티**

동기의 또 다른 기능이다.

액티비티는 세션 프로세스 사이에 논리적으로 설정되는 단위이며, 내용이 상호 독립적이다.

예를 들어 파일을 여러 개 전송한다고 할 때 각 파일은 하나의 액티비티로 처리된다.

이때도 주동기점과 부동기점이 부여된다.



#### **세션 연결**

세션 계층에서 제공하는 연결은 상위 계층의 응용 프로세스가 다른 응용 프로세스에 세션을 설정하기 위해 사용한다.



**다중 세션**

임의의 네트워크 서비스를 제공하는 서버 프로세스가 다수의 클라이언트 프로세스에 동시에 여러

세션 연결을 설정할 수 있다.

클라이언트 프로세스와 설정된 세션은 논리적으로 연관이 없는 독립적인 연결이다.

다중 세션을 지원하는 클라이언트-서버 환경은 서버가 제공하는 서비스 시간이 짧은 응용 

환경에서 유용하다. 만약 클라이언트와 서버 사이의 서비스 이용 시간이 길어지면 

세션 연결이 길게 유지되기에 다른 클라이언트의 대기 시간이 무한정 증가된다는 단점이 있다.



**단일 세션**

다중 세션 연결 방식의 문제를 해결하기 위해서는 서버 프로세스가 하나의 클라이언트와 세션 연결을

설정해 통신해야 한다. 이때 여러 클라이언트를 동시에 지원하기 위해 복사된 하위 서버 프로세스를 여러 개 만든다.



복사된 하위 서버 프로세스와 연결되기 위해서는 대표 서버 포트 주소에 연결을 시도한다.

하위 서버는 늘어나기 때문에 포트 번호를 지정하기는 어렵다.

대표 서버는 하위 서버 프로세스를 새로 실행하고 연결을 요청한 클라이언트와 세션을 연결한다.

단일 세션은 개별 요구마다 하위 프로세스를 생성하기 때문에 구축에 따른 오버헤드가 증가한다.
