**연결형 서비스**

소켓은 네트워크 통신을 위한 소프트웨어 교신점이 된다.

두 개의 독립 프로세스가 네트워크를 통해 통신하기 위해서는 논리적인 연관 관계를 맺는 소켓이 필요하다.



**-서버와 클라이언트의 동작**

서버 프로세스는 다수의 클라이언트에 공개되는 Well-known 포트로 자신의 대표 소켓 주소를 설정한 뒤

클라이언트의 연결 요청을 기다린다.

이후 클라이언트의 요구에 따라 프로세스가 제공하는 서비스가 시작된다.



**-서버의 동작 과정**

1. 서비스 교신점(호스트의 IP 주소, 포트 번호) 공개
2. 클라이언트의 요청 대기
3. 클라이언트에 서비스 제공
4. 해당 클라이언트에 서비스 제공 완료
5. 다시 2로 요청 대기



**-클라이언트의 동작 과정**

1. 원하는 서비스를 제공하는 서버 확인
2. 공개된 IP 및 포트번호로 해당 서버와 연결 시도
3. 서버에 서비스 요청
4. 서비스 이용 완료



**연결형 서비스를 시스템 콜로 이해하기**



**-서버 관점**

서비스를 제공할 소켓 생성(socket)

생성된 소켓에 주소 정보를 부여(bind)

클라이언트 요청을 받기 위한 대기 큐 지정(listen)

ㄴ 서버의 자원이 한정 되어 있기에 대기열을 생성

요청 요청 대기(accept)

데이터 받음(recv)

데이터 전송(send)



**-클라이언트 관점**

소켓 생성(socket)

서버에 연결 요청(connect)

데이터 전송(send)

데이터 받음(recv)

소켓 닫음(close)

**-클라이언트**

소켓 생성
