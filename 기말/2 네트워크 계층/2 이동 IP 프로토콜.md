서비스 중단 없이 인터넷에 접속할 수 있는 기술을 다루는 프로토콜이 이동 IP 프로토콜이다.



**터널링**

이동 호스트가 자신의 고유 주소를 유지하면서 인터넷 서비스를 받기위해서는

이동하는 송수신 호스트 간의 데이터 라우팅 처리가 중요



이동 IP 프로토콜을 이해하기 위해서는 터널링의 원리를 알아야 한다.

통신 주체가 움직인다는 것을 그룹내에서 움직이는 것으로 이해하자

하나의 그룹 내에서 다른 그룹으로 데이터를 주고 받는 것이 이동 IP 프로토콜의 핵심 원리이다.



**섬과 육지**

섬과 육지에 있는 사람들이 데이터를 주고 받는 상황을 가정해보자

섬과 육지에서는 버스를 통해 데이터를 운반할 수 있지만 섬과 육지를 가로지르기 위해서는

배라는 운반 수단이 필요하다.

버스와 배의 구조와 처리 방식은 각각 다르기 때문에 프로토콜로 이를 구현할 때는

버스를 통째로 배에 올려서 데이터를 전달하는 식으로 한다.

이렇게하면 버스에 탄 승객은 한번도 내리지 않고 원하는 목적지까지 도착할 수 있는 것이다.



**IP 터널링**

당연하지만 컴퓨터 네트워크 환경에서 호스트를 연결해 통신하기 위해서는 주소 개념이 필요하다.

기존 체계는 호스트가 고정된 위치에서 통신하는 유선 고정망 환경이기에 이동이라는 개념을 적용하는 것이

어려웠다.



**-이동**

기존 인터넷 환경의 주소 체계로 이동을 구현하는 방법은 2가지가 있다.

하나는 이동한 위치에서 새로운 IP주소를 할당하는 방식과

두번째는 호스트 고유의 IP 주소를 유지하는 방식이다.



무선 호스트가 이동할 때 발생하는 데이터 경로 문제를 프로토콜의 기본 토대를 바꾸지 않고

해결하기 위해서는 새로운 위치를 관장하는 장치인 Foreign Agent FAnew로부터 COA를 얻어야 한다



원래의 주소에서 새로운 주소로 주소가 이전되는데 이때 새로운 주소를 Foregin Network라고 한다.

따라서 새로운 주소를 관장하는 장치인 FA에게서 임시로 사용할 주소인 COA를 할당 받는다.

COA를 할당받은 뒤 바뀐 주소를 기존 주소를 관장하던 HA(Home Agent)에게 알려준다.

그리고 HA는 과거의 주소가 아닌 바뀐 주소로 데이터를 보내준다.



HA는 이동 호스트가 소속된 네트워크에서 부여하는 변하지 않는 고유 IP이며

위의 가정에서 변하는 주소는 과거 주소인 FAold와 새로운 주소인 FAnew이다.



호스트 이동 유무를 떠나 데이터는 항상 HA를 거친다.

1. 호스트가 이동한다.
2. HA가 알고 있는 내 주소와 실제 위치가 다르기 때문에 바뀐 네트워크를 관장하는 FA에게 임시 주소 COA를 받는다.
3. 바뀐 주소를 고유한 주소를 가지는 HA에게 알린다.
4. 데이터를 보낼 때는 FA가 바로 목적지로 데이터를 보내준다.
5. 만약 상대 호스트도 이동 호스트라면 목적지는 상대 호스트의 HA가 된다.
