데이터 링크 계층과 특징이 유사하지만 데이터 링크 계층은 

물리적인 전송 선로로 연결된 호스트간의 통신을 담당하고,

전송 계층은 네트워크 끝단에 위치하는 통신 주체가 논리적인 선로(네트워크)를 통해 데이터를 주고 받는다.



**차이점**

데이터 링크는 물리적인 선로로 데이터를 전달하기 때문에 중개 기능이 필요 없다.

하지만 전송 계층은 중간에 위치한 논리적 네트워크 중개 기능을 사용해 전송 기능을 수행한다.



일반적으로 기억 장치는 데이터를 읽어버리지 않게 보관하는 곳이지만 

데이터를 저장했다가 나중에 다시 사용하므로 시간적으로 지연되는 곳이라는 역할도 한다.



네트워크에서는 지연이 중요하기에 네트워크를 통해 데이터가 전송되면 송수신 호스트 사이에 지연이 발생하고

중간 네트워크가 전송 과정에서 데이터를 보관하는 효과가 발생하기도 한다.

따라서 네트워크 자체가 일종의 기억 장치 역할을 하며, 잠재적 기억 장치라고 표현한다.



**주요 기능**

**-흐름 제어**

전송 계층의 서비스를 이용해 연결을 설정하면 양 끝단의 호스트에서 실행되는 네트워크 프로세스가

데이터를 주고 받을 수 있다. 이 과정에서 필요한 주요 기능 중 하나가 흐름 제어이다.



데이터 링크 계층에서도 라우터 사이의 프레임 전송 과정에서 진행되는 흐름 제어가 있지만

라우터를 연결하는 물리적인 선로는 전송 계층이 관여하는 논리적인 네트워크 상의 호스트 사이의

논리적인 연결의 수보다 상대적으로 적다.

따라서 라우터가 논리적인 네트워크로 연결된 호스트 간의 흐름 제어 기능 외의 추가적인 기능이 필요한 것이다.

전에 배운 슬라이딩 윈도우 방식에서 송신 호스트가 보낼 수 있는 패킷의 한계를 지정하는 방식으로 문제를 해결한다.



**-오류 제어**

오류에는 데이터 변형과 데이터 분실이 있다. 변형이 발생하면 재전송으로 의한 오류 제어 기능에 의해 

복구 절차가 진행된다.



데이터 분실의 경우 각 계층의 소프트웨어가 동작하는 과정에서 데이터를 분실하는 경우가 대부분이다.



**-분할과 병합**

상위 계층에서 요구한 데이터의 크기가 전송 계층에서 처리할 수 있는 데이터의 크기보다 크면 분할을 하고

수신 프로세스가 수신한 데이터를 원래 크기로 다시 모으는 것을 병합이라 한다.



**-서비스 프리미티브**

전송 계층 사용자가 전송 계층 서비스를 사용하기 위한 인터페이스이다.



**설계 시 고려 사항**

**-주소 표현**

TCP/IP 환경에서 사용하는 호스트의 IP 주소와 포트 번호의 조합은 네트워크 계층과 전송 계층에서

사용하는 주소 표현 방식의 하나이다. 보통 전송 계층의 주소를 **TSAP**라 한다.

Transport Service Access Point



**-멀티플렉싱**

컴퓨터에는 다수의 통신 포트가 존재한다.

프로세스가 포트를 이용해 통신하기에 네트워크 프로세스는 포트 번호로 구분된다.



개별적으로 설정된 전송 계층 연결에서 전송 데이터의 단위인 TPDU의 목적지가 동일한 호스트이면

이들 데이터를 하나의 가상 회선에 실어 전송하는 것이 유리할 수 있다.



멀티플렉싱은 2가지로 구분되는데 하나는 상방향 멀티플렉싱이고 나머지는 하방향 멀티플렉싱이다.



**-상방향 멀티플렉싱**

다수의 전송 계층 연결에 대한 하부의 네트워크 계층에서 연결이 하나 형성된다.

따라서 여러 전송 계층의 연결에서 발생한 데이터가 동일한 경로로 전송되면 하나의 네트워크 연결을 묶어

전송할 수 있다.

일반적인 연결 구조보다 네트워크 계층에서 만들어지는 가상 회선 연결의 개수를 줄일 수 있기에

연결 설정에 걸리는 시간이 단축된다.



&#x09;**보충 설명**

&#x20;    송신 호스트가 각자 다른 포트에서 데이터를 같은 수신 호스트에게 전달하는 것을 예시로 들자

&#x20;    이때는 목적지가 같기 때문에 여러 포트를 사용하기 보다는 하나의 가상 회선에 실어서 보내는 것이 좋다.

&#x09;이렇게 여러 데이터 전송 흐름을 하나로 묶는 것을 상방향 멀티플렉싱이라고 한다.

&#x09;즉, 포트의 개수는 동일하지만 보내는 선로가 적어진다는 의미이다.



**-하방향 멀티플렉싱**

상방향 멀티플렉싱과는 반대로 소켓이 전송해야 하는 데이터의 용량이 큰 경우 선로를 분산해서 데이터를

전송하게 된다. 이것이 하방향 멀티플렉싱이다.



**-연결 설정**

호스트 간 통신 가능 여부를 확인하고 데이터 전송을 위한 준비를 하는 것이 연결이다.

연결을 위해 3-way handshake를 통해 호스트 간 통신 준비를 하게 된다.

1. A프로세스가 임의의 송신 순서 번호 x를 지정해 응답을 보낸다. / Conn\_Req(x)
2. B프로세스가 x가 담긴 Conn\_Req(x)를 받고 본인의 순서 번호 y를 담아서 보낸다. / Conn\_Ack(y,x)
3. 연결 요구에 대한 Conn\_Ack(y,x)를 받은 A는 이에 대한 응답을 하고 데이터를 주고 받는다. / Data\_Req(x,y)



**-연결 해제**

일방적 연결 해제 절차 방식에서는 한쪽 프로세스가 일방적으로 Disc\_Req를 전송해 연결 해제를 한다.

일방적이기에 상대 프로세스의 동의가 없이도 연결이 끊기는 방식이다.

이 경우 상대가 보낸 패킷이 아직 네트워크에서 전달되지 못한 경우에도 연결이 해제된다.

이른바 유령 패킷 문제가 발생한다는 것이다.



점진적 연결 해제 절차 방식은 두 프로세스가 서로 각각의 데이터 전송 소켓을 가진다.

한쪽이 일방적으로 소켓을 해지해도 남은 한 쪽의 소켓이 해지되지 않는다는 의미이다.

이렇게 함으로서 전달되지 못한 데이터가 네트워크 상에 남아있더라도 안전하게 데이터가 전달될 수 있다.

점진적 연결 해제 절차 방식에서 연결을 해제하기 위해서 두 프로세스 모두 Disc\_Req를 전송해야 한다.
