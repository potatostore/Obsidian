TCP는 IP 프로토콜위에서 연결형 서비스를 지원하는 프로토콜이다.

연결형 서비스를 주로 제공하고 전이중 방식의 양방향 가상 회선을 제공한다.

가장 주요한 특징으로는 신뢰성을 보장한다는 것이다.



**헤더 구조**

Source Port, Destination Port

Sequence Number, Acknowledgement Number



SP,DP: TCP로 연결되는 가상 회선 양단의 프로세스에 할당된 네트워크 포트 주소

Sequence Number: 송신 프로세스가 지정하는 순서 번호

Acknowledgement Number: 수신 프로세스가 제대로 수신한 바이트의 수를 응답하기 위해 사용

Data Offset, Reserved(예약 필드), Window(수신 윈도우의 버퍼 크기를 지정), Checksum, Urgent Pointer(긴급 데이터 처리)



**-헤더의 플래그 비트**

URG(UP 필드 유무), ACK(AN 필드 유무), 

PSH(현재 세그먼트에 포함된 데이터를 상위 계층에 즉시 전달 요구), 

RST(연결의 리셋이나 유효하지 않은 세그먼트에 대한 응답), SYN(연결 설정 요구), FIN(연결 종료 요구 의사 표명)



**혼잡 제어**

TCP 프로토콜에 추가된 ECN 기능은 라우터가 송신 프로세스에 명시적으로 혼잡 발생을 알리는 역할을 한다.

이를 위해 CWR 필드와 ECE 필드를 정의한다.

**-CWR**

ECE 비트를 수신한 송신 프로세스가 송신 윈도우 크기를 줄였음을 통지하는 것이 목적

더 이상 ECE를 전송하지 말라는 의미



**-ECE**

ECN-Echo를 의미, 네트워크 트래픽이 많아질 때 라우터가 송신 프로세스에 명시적으로 혼잡을 알릴 때 사용



**TCP 캡슐화**

전송 계층의 데이터에 TCP 헤더가 붙고 네트워크 계층으로 전달



**포트**

포트 번호는 TCP와 UDP가 상위 계층에 제공하는 주소 표현 방식이다.

책에는 없지만 송신 IP, 송신 포트, 수신 IP, 수신 포트 4쌍의 정보로 통신 호스트를 구분한다.

통념적으로 http 80, https 443이렇게 포트를 사용하는데 이런 포트들을 WELL-known 포트라고 한다.



**동작 방식**

3way-handshake로 연결되고 Sequence Number를 이용해서 각 프로세스가 보낸 데이터의 고유 번호를 명시한다.



만약 TCP 세그먼트 3개를 연속으로 전송하는데 3번째 세그먼트에 오류가 발생한 경우

오류가 발생한 세그먼트의 번호를 이용해 재전송을 요구한다.



연결을 해제할 때는 FIN 플래그를 지정해서 요구한다.

TCP의 연결 해제는 양방향으로 진행되기에 데이터 전송이 모두 마무리된 이후 두 프로세스 모두 FIN 플래그를 지정해

연결을 해제한다



**-혼잡 제어**

3way-handshake를 할 때 각 프로세스가 혼잡 제어 기능을 사용할 것인지 합의해야 한다.

만약 두 프로세스 모두 혼잡 제어 기능을 지원하고 사용을 원하는 경우

A->B로 SYN,CWR,ECE를 전달하고 다시 B->A로 SYN,ACK,ECE를 전달해 두 프로세스 모두 혼잡 제어 기능을

사용한다고 합의한다.



반대로 혼잡 제어를 원하지 않는 경우 A->B로 SYN,CWR,ECE를 전달하고

나머지는 SYN,ACK를 보내 혼잡 제어 기능을 사용하지 않는다.



이후 라우터를 이용해서 혼잡 제어를 할 때는 라우터에게 혼잡을 인지시키고

수신 프로세스가 ECE로 응답을 한 뒤 송신 호스트가 혼잡을 인지하고 송신 윈도우를 조절한다.
