
# 사용흐름

#### 사용자가 프롬프트 입력 시
1. 사용자가 찾고 싶은 사진에 대한 프롬프트를 입력 : "손에 장난감을 든 아이의 사진을 찾아줘"
2. Ondevice AI를 통해 프롬프트를 Image MyData Tag로 바꾸는 작업 실행 : <손, 장난감, 아이> 등
3. 결과물을 출력

1-A) 어플을 키거나, 매일 00:00 사진첩 확인을 통해 추가 및 삭제된 사진에 대해 태깅작업 실시
1-B) 태깅작업을 통해 어플리케이션에 할당된 스토리지에 RDB형식으로 연결 : 3개의 테이블로 구성되어 있는데, 첫 번째가 사용자가 사진을 찍고, 저장되는 사진첩, 두 번째가 어플리케이션  내 
<사진첩에 저장된 사진을 가르키는 key, Tag key>, 세 번째가 <Tag key, Tags...>
2-A) Ondevice AI를 사용해야 하기에, 사용자가 Android : Gemini Nano banana / IOS : Apple Intelligency가 초기 설정시 깔려있지 않거나, 위 1-A) 업뎃 시 깔려있지 않으면 까는 것을 알림으로 권장
3-A) 결과물 출력시 단순 태그 출력이 아닌, 위 1-B)에서 첫 번째 사진첩에 접근하여 해당 사진을 출력해주는 형식으로(따라서 처음 어플리케이션 설정 시 사진첩 접근 권한을 반드시 취득하는 방향으로 진행)
